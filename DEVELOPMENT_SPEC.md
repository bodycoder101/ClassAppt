# 托育园课程预约系统开发规格说明

## 1. 项目目标

在当前微信小程序课程预约项目基础上，改造为适用于托育园场景的课程预约与教务管理系统。系统需支持家长端、教师端、管理端三类角色，覆盖课程排课、预约、签到、请假补课、课后反馈、统计、课消与多园区扩展。

## 2. 角色与权限

### 家长端
- 微信登录与手机号绑定。
- 可绑定多个孩子。
- 查看课程、课表、剩余名额、老师、地点、适合年龄。
- 预约课程、取消预约、提交请假/补课申请。
- 查看签到记录、老师评语、课堂照片、学习记录、一日反馈。

### 教师端
- 查看今日课表。
- 查看上课学生名单。
- 手动签到与出勤确认。
- 上传课堂照片。
- 填写课堂反馈、学生表现、成长档案。
- 发起调课/代课申请，等待管理端审批。

### 管理端
- 管理园区、教师、班级、孩子、课程、课表。
- 审核预约、请假、补课、调课/代课。
- 处理课程变更：换老师、换教室、取消课程。
- 查看统计：出勤率、预约率、热门课程、教师工作量、课消、财务。
- 管理角色权限：园长、教务、班主任、教师、普通管理员。

## 3. 核心业务模块

## 3.1 课程与排课
- 支持园区自定义课程类型。
- 支持固定周期课、单次课、学期课三种模式。
- 课程字段至少包括：课程名称、课程类型、老师、地点、适合年龄、适用班级、课程介绍、封面、人数上限、是否需要审核。
- 排课支持按周批量生成，也支持单次手动发布。
- 节假日、园方活动日支持手动调整，并触发通知。

## 3.2 预约规则
- 满员后显示“已满”，并推荐其他可约时段。
- 支持预约审核策略：无需审核、全部审核、指定课程审核。
- 支持提前开放预约：按周、按月、指定日期。
- 支持提前一天取消。
- 记录临时取消次数，超过 3 次后按规则限制预约或扣课。
- 记录爽约，支持超过阈值后限制预约或扣课。
- 家长每周/月预约次数当前按需求暂不限制，但系统需预留规则配置。

## 3.3 请假与补课
- 家长可在线提交请假。
- 请假支持老师或管理端审批。
- 补课采用手动申请、人工审批模式。
- 补课时段从可预约课表中选择。
- 请假、补课、审批结果需写入孩子课程记录。

## 3.4 签到与课后反馈
- 教师端支持手动勾选签到。
- 预留自动签到能力。
- 课后反馈包括课堂反馈、照片、学生表现、出勤情况、成长档案。
- 托班增加一日情况反馈。

## 3.5 通知
- 预约成功通知。
- 开课前提醒。
- 课程变更通知。
- 请假/补课审批结果通知。
- 活动通知。
- 剩余课程提醒。
- 临时取消、请假补课规则提醒。
- 第一阶段优先接入微信订阅消息，短信作为扩展能力。

## 3.6 课消、支付与合同
- 支持课程包、次卡、月卡。
- 预约到课后产生课消记录。
- 爽约可按规则自动扣课。
- 支持订单、支付、退款、财务汇总。
- 支持电子合同记录与查看，签署能力可作为后续扩展。

## 3.7 多园区与系统对接
- 支持多园区数据隔离。
- 用户、孩子、课程、订单、课消均需归属园区。
- 支持学员名单和课程数据导入。
- 预留外部托育管理系统接口。

## 4. 建议数据模型

- `campus`：园区信息。
- `teacher`：教师信息、所属园区、任课能力。
- `child`：孩子档案、年龄、班级、家长绑定关系。
- `class`：班级信息、班主任、园区。
- `course`：课程定义。
- `schedule`：具体课表与时段。
- `booking`：预约记录，可由现有 `join` 演进。
- `attendance`：签到与出勤记录。
- `leave`：请假申请。
- `makeup`：补课申请。
- `feedback`：课堂反馈、照片、成长记录。
- `package`：课程包、次卡、月卡。
- `order`：订单、支付、退款。
- `course_consume`：课消记录。
- `role_permission`：角色权限配置。

## 5. 当前项目复用点

- 保留微信小程序端框架、云函数框架、云开发调用封装。
- 保留预约列表、预约详情、预约日历、我的预约等基础页面逻辑。
- 保留动态表单能力，用于补充报名字段。
- 保留资讯/通知内容管理基础能力。
- 保留后台页面结构，但需补齐被关闭的后端写操作。

## 6. 验收标准

MVP 验收需至少满足：
- 管理端可创建课程、配置老师/地点/年龄段/时段/人数上限。
- 家长可绑定多个孩子并完成预约。
- 教师可查看今日课表、学生名单并完成签到。
- 家长可取消预约、提交请假。
- 管理端可查看预约名单、出勤记录、基础统计。
- 关键通知可通过微信订阅消息发送。
- 核心流程在微信开发者工具和真机预览中通过验证。
